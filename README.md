# 🛒 LZT Market Bot — карманный маркетплейс аккаунтов

**Telegram-бот для полного управления товарами на [LZT Market](https://lzt.market)
прямо из мессенджера.** Создание, просмотр, изменение цен, открытие/закрытие,
удаление и экспорт аккаунтов — всё, что обычно делается в веб-кабинете Маркета,
теперь помещается в карман.

Написан на `aiogram 3` поверх официального
[LZT Market API](https://lzt-market.readme.io/) и полностью соответствует его
спецификации (OpenAPI 3.1).

---

## 🎯 О проекте

Продавцу аккаунтов неудобно каждый раз заходить в веб-кабинет ради рутины:
залить товар, подрезать цены под конкурентов, закрыть/открыть партию лотов.
Этот бот превращает управление магазином в пару нажатий в Telegram:

- **Залив за минуты** — пошаговый мастер под **24 категории** (Steam, Fortnite,
  Epic Games, Telegram, Supercell, WoT, VPN, miHoYo, TikTok, Discord, Roblox…),
  каждая со своими полями: тип почты, куки, регион, сервис, гарантия и т.д.
- **Управление на ходу** — карточка каждого товара, массовые операции,
  умная смена цен (снизить на % или задать вручную по ссылкам).
- **Фирменный вид** — цвета кнопок и кастом-эмодзи из наборов Telegram
  (Bot API 9.4), тёмная тема с зелёным акцентом Маркета.

## ✨ Возможности

| Раздел | Что умеет |
|--------|-----------|
| 📦 **Создание товара** | Мастер под 24 категории: происхождение, тип почты (родная / авторег / временная от Маркета), длительность гарантии, куки (частями или файлом `.json`), Telegram через `session.json`. Двухшаговая публикация `item/add` → `goods/check`, открытым или закрытым. |
| 📊 **Мои товары** | Список с постраничной навигацией, фильтр активные/закрытые, карточка товара. |
| 🎛 **Карточка товара** | Изменить цену • Открыть/Закрыть • Удалить (с подтверждением). |
| ⚙️ **Массовые операции** | Смена цен (**снизить на %** или **задать вручную** по ссылкам), открыть/закрыть/удалить закрытые, экспорт ссылок в `.txt`. Работают по всем страницам. |
| 👤 **Профиль** | Имя, баланс, холд и статистика по товарам (`/me`). |

## 🧰 Что нужно для запуска

- **Python 3.10+**
- Зависимости из `requirements.txt` (`aiogram 3.29`, `aiohttp`, `pydantic`, `python-dotenv`)
- **Токен Telegram-бота** — от [@BotFather](https://t.me/BotFather)
- **Токен LZT Market API** — lzt.market → Settings → API

Для кастом-эмодзи в кнопках/тексте нужен **Telegram Premium** у владельца бота
(или Fragment-юзернейм). Без него всё работает — просто с обычными эмодзи.

## 🚀 Запуск (одна команда)

Склонируйте репозиторий и запустите `run.py` — он сам поставит зависимости,
спросит токены в консоли, сохранит их в `.env` и запустит бота:

```bash
pip install -r requirements.txt
python run.py
```

При первом запуске в консоли появится:

```
🔑 Токен Telegram-бота — получите у @BotFather
   BOT_TOKEN = <вставьте токен>
🔑 Токен LZT Market API — lzt.market → Settings → API
   LOLZTEAM_TOKEN = <вставьте токен>

✅ Настройки сохранены в .env
🚀 Запуск бота...
```

После ввода токенов бот запускается и работает. При следующих запусках токены
уже сохранены — `python run.py` просто поднимает бота.

### Ручной способ (по желанию)

```bash
pip install -r requirements.txt
cp .env.example .env   # впишите токены
python bot.py
```

### Переменные окружения (`.env`)

| Ключ | Описание |
|------|----------|
| `BOT_TOKEN` | Токен Telegram-бота от @BotFather |
| `LOLZTEAM_TOKEN` | Bearer-токен LZT Market API |
| `API_BASE_URL` | База API, по умолчанию `https://prod-api.lzt.market` |

> ⚠️ `.env` с реальными токенами не коммитится — он в `.gitignore`.

## 🗂 Структура

```
run.py             — установщик: зависимости + токены + запуск (одна команда)
bot.py             — точка входа, polling, меню команд
config.py          — конфиг и лимиты запросов
api_client.py      — async-клиент LZT Market API (rate-limit + retry + пагинация)
forms.py           — мастер создания товара, поля 24 категорий
item_management.py — просмотр списка, карточка, управление отдельным товаром
handlers.py        — команды, массовые операции, профиль, регистрация хендлеров
keyboards.py       — inline-клавиатуры (цвета + кастом-эмодзи)
ui.py              — единый визуальный стиль
data/user_states.py— модели состояния
```
